Prerequisites

  • Python 3.10+ (3.14 recommended)
  • Node.js 18+ for frontend build
  • Git for version control
Windows Note If your Python path contains spaces (e.g., C:\Users\Ravi Hegde\...), always use python manage.py start instead of Start-Process.

Quick Start

Start the server in background mode:

python manage.py start

Then open http://localhost:8080 in your browser.

Setup Wizard

On first launch, HomeGallery guides you through a multi-step setup wizard.

1

Photo Library Path

Configure the directory containing your photos. The server will scan and index all images recursively.

  • Supported formats: JPEG, PNG, WebP, GIF
  • Max file size: 50MB (configurable)
  • Recursive subdirectory scanning
  • Safe path validation prevents directory traversal
Step 1: Photo Library
2

Admin Account

Create your admin account for managing the gallery. All subsequent access requires authentication.

  • Password: minimum 6 characters, mixed case + numbers
  • JWT token expiry: 24 hours (configurable)
  • Passwords: bcrypt hashed (never stored in plain text)
  • Rate limiting: 5 attempts per 15 minutes per IP
Step 2: Admin Account
3

Server Configuration

Configure the server host and port for your HomeGallery instance.

  • Default host: 0.0.0.0 (all interfaces)
  • Default port: 8080
  • Use 0.0.0.0 to allow network access
Step 3: Server Configuration
4

Database

Choose your database backend. SQLite is recommended for single-user setups.

  • SQLite: Local, zero configuration
  • PostgreSQL: Remote, for scaling
  • Database URL configurable
Step 4: Database
5

Background Processing

Configure automatic processing options for thumbnails and face detection.

  • Auto-generate thumbnails on upload
  • Background face detection (requires more memory)
  • All options changeable later in Settings
Step 5: Background Processing
6

Review & Save

Review your configuration before saving. All settings can be edited later via config.json.

  • Photo directory path
  • Admin username
  • Server host and port
  • Database type
Step 6: Review Configuration

API Key Management

HomeGallery supports external AI providers for cloud-based analysis. API keys are encrypted at rest using Fernet symmetric encryption with a SHA-256 derived key from your SECRET_KEY.

Security Keys are encrypted before storage. Only masked versions are shown in the UI and API responses. Admin-only access required via get_current_admin_user dependency. No logging of API key values.

Supported Providers

Provider Use Case Key Format
OpenAI ChatGPT image analysis sk-proj-...
Google Gemini Gemini vision model AIzaSy...
OpenRouter Multi-model routing sk-or-v1-...
Anthropic Claude vision sk-ant-...

Managing Keys

Go to Settings > API Keys to add, enable/disable, or delete API keys. Keys are validated on save and encrypted immediately.

API Endpoints

GET    /api/api-keys          # List all keys (admin only)
GET    /api/api-keys/{id}     # Get single key (admin only)
POST   /api/api-keys          # Create new key (admin only)
PUT    /api/api-keys/{id}     # Update key (admin only)
DELETE /api/api-keys/{id}     # Delete key (admin only)

Agent System

5 autonomous agents run background processing jobs. All disabled by default. Enable via Settings > Agents.

Agent Interval Function
Smart Metadata Every 5 min EXIF, objects, colors, tags extraction
Auto-Organization Every 15 min Date albums, GPS clusters, duplicate detection
Enhancement Every 10 min Histogram analysis, scene presets, quality scoring
Content Analysis Every 10 min Sharpness, exposure, noise, composition metrics
Visual Search Every 10 min Feature extraction, embeddings, similarity search
